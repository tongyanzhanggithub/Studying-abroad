## 启动

```bash
npm run start:local
```

一条命令把数据库和网站一起起来,Ctrl+C 一起停。浏览器开 `http://localhost:3000`。

> **一个必须知道的限制**:本地数据库(PGlite)同一时刻**只接受一个连接**。
> 所以网站跑着的时候,`db:seed` / `data:import` / `admin:create` 这些命令会连不上 ——
> 要跑它们先 Ctrl+C 停掉网站。
>
> 卡住了就:`taskkill /F /IM node.exe`,然后重新 `npm run start:local`。
>
> 这个限制只在本地存在。服务器上用的是真的 PostgreSQL,没这个问题。

**改完代码点不动了?** 开发模式下改 server action 有时会让页面上的按钮"点了没反应"
(客户端还引用着旧的 action ID)。Ctrl+C,删掉 `.next` 文件夹,重新 `npm run start:local`。
这是 Next.js 开发环境的老毛病,不是代码问题。

## 建议的测试顺序

按真实用户路径走,比一页页翻更容易发现问题。

### A. 没登录的人(获客漏斗)

1. **首页** —— 数字对不对?文案有没有看不懂的行话?
2. **免费测一测** 走完 3 步 —— 有没有哪一步让你卡住、不知道该填什么?
3. **结果页** —— 概率区间可信吗?语言体检和申请时间那两块有用吗?
4. **定价页** —— 价格和退款规则说清楚了吗?看完你会想买吗?

### B. 付费学生(核心留存)

5. 登录 → 定价页买季票(模拟支付,不会真扣款)
6. **付完应该落在引导页**,不是四个 0 的空白总览 —— 落错了是 bug
7. **总览** —— 顶部「现在最该做的」给的三件事合理吗?为什么是这三件说清楚了吗?
8. **评估**(新)—— 多份方案能并排对比吗?「按我现在的资料重算」告诉你的变化有用吗?
9. 回结果页 —— 会员应该看到**完整名单**且**没有付费引导**,还能一键导入选校单
10. **选校** —— 卡片上的信息够你判断"加不加"吗?加入是不是一步到位?
11. **材料** —— 传个文件,**点开确认打得开**(这里之前是个黑洞,刚修)
12. **文书 / 服务 / 订单** —— 退款金额算得对不对

## 怎么把问题给我

一句话说清这三件事就够:

```
在哪一页 + 我做了什么 + 我期望看到什么 / 实际看到什么
```

例子:

> 在结果页,我填的是双非 78 分,结果一个学校都没匹配到。
> 我以为至少会给几个保底的。

> 后台人工服务页,我把「全程陪跑包」改成 9800,保存后前台还是 12800。

**界面丑 / 文案别扭也算问题**,直接说「这句话我看不懂」或者「这一屏太挤」就行,
不用想怎么改 —— 你上次说"内容和术语太难看",那次反馈的价值比我自己改十版都大。

## 改完代码要跑的检查

```bash
npm run check    # typecheck + 生产构建
```

**只跑 `npm run typecheck` 不够。** 这个项目已经四次栽在「tsc 通过但 `next build` 失败」上,
而部署脚本里跑的正是 `next build` —— 漏掉就是部署当场失败。

另外三个自检(不需要真实 LLM key):

```bash
curl -X POST -H "x-cron-secret: $AUTH_SECRET" localhost:3000/api/dev/verify-collect     # AI 采集防编造
curl -X POST -H "x-cron-secret: $AUTH_SECRET" localhost:3000/api/dev/verify-dispatch    # 派单状态机
curl -X POST -H "x-cron-secret: $AUTH_SECRET" localhost:3000/api/dev/verify-region-gate  # 地区闸门
```
