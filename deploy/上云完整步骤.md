## 步骤 2 · 装环境(加 swap / Postgres / Node / Nginx)

```bash
cd /opt/compass && sudo bash deploy/setup-server.sh
```

**为什么必须先加 swap**:这台机只有 2 GiB,Next.js 生产构建峰值要 1.5–2 GB。
不加 swap 会在 `npm run build` 时被 OOM Killer 杀掉,而报错常常**只有一行 `Killed`**,
很难看出是内存问题,容易白折腾半天。

**验证**:

```bash
free -h | grep -i swap && node -v && nginx -v && systemctl is-active postgresql
```
期望:swap 有 4.0Gi、Node v20+、nginx 版本号、postgresql 是 `active`。

## 步骤 3 · 建库 + 生成 .env

```bash
cd /opt/compass && sudo bash deploy/setup-db.sh
```

它会建库建账号,并生成 `/opt/compass/.env`(权限 600),里面:
- `DATABASE_URL` —— 随机强密码
- `AUTH_SECRET` —— `openssl rand -base64 32` 现生成
- `NEXT_PUBLIC_SITE_URL` —— 自动取本机公网 IP
- 支付/短信/LLM/存储 —— 全是 `mock`(资质没下来之前就该是这样)

**验证**:

```bash
ls -l /opt/compass/.env && grep -c '^DATABASE_URL=' /opt/compass/.env
```
期望:权限 `-rw-------`,`DATABASE_URL` 计数为 1。

> ⚠️ **别把 `AUTH_SECRET` 留成开发默认值**。应用启动时有自检,
> 如果它还是 `dev-only...` 或 `NEXT_PUBLIC_SITE_URL` 还是 localhost,
> **会直接拒绝启动**(这是有意的:带着这种配置跑没人会发现,直到出事)。

## 步骤 4 · 部署(装依赖 / 建表 / 导数据 / 构建 / 起服务)

```bash
cd /opt/compass && sudo bash deploy/deploy.sh
```

最慢的一步,2G 机器可能要 5–10 分钟。它会:
装 npm 依赖 → `prisma db push` 建表 → `NODE_ENV=production npm run db:seed` 灌套餐/规则
→ `npm run data:import` 导入 310 个院校项目 → `npm run build` → 配 Nginx → 起 systemd 服务 → 写定时任务。

**验证**:

```bash
systemctl is-active compass && curl -s -w " %{http_code}\n" http://127.0.0.1:3000/api/health
```
期望:`active`,以及 `{"ok":true} 200`。

> `/api/health` 会**真的查一次数据库**,不是只返回 200 ——
> 「进程活着但连不上库」这种情况首页照样能开(有兜底数据),只有它能发现。

## 步骤 5 · 打开演示登录开关(**学生端能登录的前提**)

生产环境 + mock 短信,默认是**直接报错拒绝登录**的(不能假装成功,否则用户干等一条永远不到的短信)。
演示环境要能走学生流程,显式打开:

```bash
cd /opt/compass && echo 'ALLOW_MOCK_SMS="true"' >> .env && systemctl restart compass
```

打开后**验证码只写进服务端日志,绝不返回给页面** —— 所以只有能 SSH 看日志的人能登,
公网上的人拿不到别人的码。

> ⚠️ **接入阿里云短信后,必须把这一行删掉。**

## 排查:502 / 起不来

```bash
systemctl status compass --no-pager -l
journalctl -u compass -n 60 --no-pager
```

| 日志关键字 | 原因 | 修 |
|---|---|---|
| `缺少必需的环境变量 DATABASE_URL` | `.env` 没生成 | 重跑步骤 3 |
| `❌ 生产环境配置有致命问题,拒绝启动` | 启动自检拦下了 | 按提示改 `.env` 对应项 |
| `Killed` / 构建中途没了 | **OOM**(最常见) | `free -h` 看 swap;没有就重跑步骤 2 |
| 找不到 `.next` / `Cannot find module` | 构建没成功 | 重跑步骤 4 |
| `ECONNREFUSED ... 5432` | Postgres 没起 | `systemctl restart postgresql` |
| Nginx 502 但 `curl 127.0.0.1:3000` 通 | Nginx 配置问题 | `nginx -t && systemctl reload nginx` |

**改完 `.env` 一定要重启**:`systemctl restart compass`
